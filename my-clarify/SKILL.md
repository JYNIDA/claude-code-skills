# EO Clarify: 콘텐츠 기획 요구사항 명확화

EO Global 콘텐츠 기획 과정에서 모호하거나 애매한 요구사항을 가설 기반 질문으로 구체적인 실행 스펙으로 바꾼다.
**반드시 AskUserQuestion 도구를 사용**한다 — 일반 텍스트로 질문하지 않는다.

## 언제 사용하는가

- 애매한 영상 기획 요청 ("이 사람 영상 한번 만들어보자")
- 불완전한 게스트 제안 ("요즘 뜨는 사람 찾아줘")
- 구체적이지 않은 콘텐츠 방향 ("AI 관련 영상 하나 더 하자")
- 시리즈/포맷 기획 ("새로운 시리즈 하나 만들까")

## 핵심 원칙: 가설을 선택지로

열린 질문 대신, 그럴듯한 해석을 선택지로 제시한다.
각 선택지는 "사용자가 실제로 원하는 것"에 대한 검증 가능한 가설이다.

```
BAD:  "어떤 영상을 만들고 싶으세요?"          ← 열린 질문, 높은 인지 부하
GOOD: "창업 스토리 / 기술 해설 / 투자 인사이트" ← 하나 고르기, 낮은 부하
```

## 프로토콜

### Phase 1: 원본 캡처 + 진단

원래 요구사항을 그대로 기록한다. 모호한 부분을 식별한다:
- 게스트/주제가 명확한가?
- 영상의 앵글(관점)이 정해져 있는가?
- 타겟 시청자가 누구인가?
- 바이럴 포인트가 있는가?

### Phase 2: 반복 질문 (AskUserQuestion)

AskUserQuestion으로 모호함을 해소한다. **관련 질문을 최대 4개까지 묶어서** 한 번에 물어본다.

**상한: 5-8개 질문.** 핵심 모호함이 해소되거나, 사용자가 "됐어"라고 하거나, 상한에 도달하면 멈춘다.

**모호함 카테고리**:

| 카테고리 | 가설 예시 |
|----------|----------|
| **게스트** | 특정 인물 / 카테고리(AI CEO 등) / 트렌드 기반 탐색 |
| **앵글** | 창업 스토리 / 실패담 / 기술 비전 / 논쟁적 관점 |
| **타겟 시청자** | 예비 창업자 / 현직 CEO / 테크 관심 일반인 / 투자자 |
| **바이럴 포인트** | 충격적 숫자 / 반전 스토리 / 유명인 언급 / 시의성 |
| **형식** | 롱폼 인터뷰 / 숏폼 클립 / 다큐 스타일 / 패널 토론 |
| **긴급도** | 이번 주 촬영 / 이번 달 / 분기 내 / 시리즈 기획 |
| **언어** | 영어 인터뷰 / 한국어 자막 병기 / 한국어 더빙 |

### Phase 3: Before/After 요약

변환 결과를 보여준다:

```markdown
## 콘텐츠 기획 Clarify 결과

### Before (원본)
"{원래 요구사항 그대로}"

### After (구체화됨)
**게스트/주제**: [누구 또는 무엇에 대한 영상인지]
**앵글**: [이 영상만의 뾰족한 관점]
**타겟 시청자**: [누가 볼 영상인지]
**바이럴 포인트**: [왜 지금 이 영상이 터질 수 있는지]
**형식**: [롱폼/숏폼/시리즈 등]
**긴급도**: [언제까지]

**결정 사항**:
| 질문 | 결정 |
|------|------|
| [모호했던 점 1] | [선택한 옵션] |
```

### Phase 4: 저장

결과를 파일로 저장할지 물어본다. 기본 위치: `~/content-pipeline/clarify/`

## 규칙

1. **가설로 물어본다**: 모든 선택지는 그럴듯한 해석이다
2. **가정하지 않는다**: 추측 대신 질문한다
3. **의도를 보존한다**: 정제하되 방향을 바꾸지 않는다
4. **5-8개 질문이 한계다**: 그 이상은 피로감
5. **관련 질문은 묶는다**: AskUserQuestion 한 번에 최대 4개
6. **변화를 추적한다**: 반드시 Before/After를 보여준다
7. **바이럴 관점을 포함한다**: "왜 이 주제가 지금 터질 수 있는지" 항상 고려한다
