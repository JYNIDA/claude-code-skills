# EO Global Interview Research Workflow v1

## 인터뷰이 리서치 → 주제 선정 → 질문지 작성

**EO PRODUCTION TEAM**
**Version 1.0 | 2026년 3월**

> 이 문서는 EO Global 인터뷰 콘텐츠의 사전 리서치 표준 프로세스를 정의한다.
> Copy Workflow v5 (제목/썸네일)는 촬영 후 편집 단계에서 사용하는 문서이고,
> 이 문서는 **촬영 전** — 게스트 확정부터 질문지 완성까지를 다룬다.

---

## 워크플로우 전체 흐름

```
Phase 0 ── 입력: 게스트 이름 + 링크 (LinkedIn, 웹사이트 등)
    │
Phase 1 ── 게스트 리서치 (WHO is this person?)
    │        1.1 프로필 빌딩
    │        1.2 콘텐츠 감사 (이 사람이 만든/나온 콘텐츠 전수 조사)
    │        1.3 기존 인터뷰 분석 (다른 채널에서 뭘 물어봤나)
    │        1.4 히든 스토리 발굴 (대부분이 모르는 것)
    │
Phase 2 ── 주제 선정 (WHAT should we talk about?)
    │        2.1 Hot Take 추출
    │        2.2 해묵은 얘기 필터링
    │        2.3 Theme + Title/Thumbnail + Lessons 구조화
    │        2.4 EO 채널 적합성 체크
    │
Phase 3 ── 질문지 작성 (HOW do we get it?)
    │        3.1 Lesson별 질문 설계
    │        3.2 Follow-up & 디플렉션 대응
    │        3.3 팩트체크 레퍼런스
    │        3.4 프로덕션 노트
    │
Output ── interview_prep_{guest}_{date}.md
```

---

## Phase 0: 입력

워크플로우 시작에 필요한 최소 정보:

```
■ 게스트 이름:
■ 링크: (LinkedIn, 개인 웹사이트, 회사 등 아는 만큼)
■ 시리즈: (예: Thinking Mode, Before 25, Intelligence 등)
■ 왜 이 사람인가: (섭외 배경 — 한 줄이라도)
```

> 이 정보만 있으면 Phase 1을 시작할 수 있다.

## Phase 1: 게스트 리서치

### 1.1 프로필 빌딩

**목표:** 이 사람이 누구인지 360도 파악.

**수집 항목:**

| 항목 | 설명 | 필수 |
|------|------|:----:|
| **기본 정보** | 이름, 현재 직함, 회사, 나이(공개된 경우) | ✅ |
| **경력 타임라인** | 시간순 주요 이력. 표 형식. | ✅ |
| **핵심 성과** | 이 사람을 유명하게 만든 것 (수치 포함) | ✅ |
| **교육 배경** | 학교, 전공 (공개된 경우) | ⚪ |
| **링크** | LinkedIn, X, 웹사이트, YouTube, 뉴스레터 등 | ✅ |
| **유명도 판단** | 대중적 유명 / 업계 유명 / 무명 | ✅ |

**유명도 판단 기준 (Copy Workflow v5 연동):**

| 유명도 | 판단 기준 | 제목 전략 |
|:---:|-----------|-----------|
| 대중적 유명 | 비테크 사람도 아는 이름 (일론 머스크급) | 이름이 곧 훅 |
| 업계 유명 | 테크/빌더 커뮤니티에서 알려진 인물 | 이름은 파이프 뒤. 스토리/인사이트가 메인 |
| 무명 | 업적은 있지만 이름 인지도 없음 | 이름 안 씀. 숫자/스토리로 훅 |

**소스 우선순위:**

| 순서 | 소스 | 이유 |
|:----:|------|------|
| 1 | 개인 웹사이트 / About 페이지 | 본인이 원하는 내러티브 파악 |
| 2 | LinkedIn | 경력 팩트 확인 |
| 3 | 주요 언론 프로필 기사 | 외부 시선 + 검증된 수치 |
| 4 | X/Twitter, Threads | 최근 관심사, 비공식 발언 |
| 5 | 뉴스레터/블로그 | 깊은 사고 구조 파악 |
| 6 | 팟캐스트/유튜브 출연 | 말하는 스타일, 반복 패턴 파악 |

---

### 1.2 콘텐츠 감사 (Content Audit)

**목표:** 이 사람이 최근 무슨 이야기를 하고 있는지, 어떤 프레임워크/주장을 갖고 있는지 파악.

**체크리스트:**

- [ ] 최근 3개월 이내 글/포스트/영상 목록 수집
- [ ] 핵심 프레임워크나 반복 주장 추출 (예: "5 Levels of AI Native")
- [ ] 최신 관심사 키워드 정리 (예: AI agents, MCP, prototype-first)
- [ ] 뉴스레터 구독자 수, YouTube 구독자 수 등 영향력 수치
- [ ] 유료 상품이 있는 경우 (강의, 커뮤니티 등) 확인

**출력 형태:**

```
최근 콘텐츠 키워드: [AI native, agents, prototype-first, ...]
핵심 프레임워크: [5 Levels of AI Native, Build for Agents First, ...]
최근 트렌드: [Claude Code 튜토리얼, OpenClaw, Google AI Studio, ...]
```

### 1.3 기존 인터뷰 분석

**목표:** 다른 인터뷰어들이 이미 물어본 것을 파악하고, EO에서는 다른 앵글을 잡는다.

**분석 표:**

| 소스 | 게스트 모드 | 뭘 배웠나 | EO에서 쓸 수 있나 |
|------|------------|----------|:---:|
| (미디어명, 날짜) | (pitch mode / personal / thought leader 등) | (핵심 내용) | ✅/✗ |

**핵심 분석 포인트:**

1. **반복 스토리 탐지** — 3회 이상 나온 이야기는 "이미 소진된 스토리"
2. **모드 패턴 탐지** — 이 사람이 어떤 질문에 어떤 모드로 대답하는지
   - 감정 질문 → 숫자로 도피?
   - 개인 질문 → 프레임워크로 도피?
   - 구체적 질문 → 일반론으로 도피?
3. **수치 불일치 탐지** — 같은 수치가 소스마다 다르면 기록 (리허설된 스토리의 자연 드리프트)
4. **아직 안 물어본 것** — 모든 인터뷰어가 빠뜨린 질문이 EO의 기회

**이미 소진된 질문 목록 (DO NOT RE-ASK):**

```
- (여기에 이전 인터뷰에서 반복된 질문들 나열)
```

### 1.4 히든 스토리 발굴

**목표:** 대부분의 사람이 모르는 이야기를 찾아서, EO 인터뷰에서만 들을 수 있는 내용을 만든다.

**히든 스토리 소스:**

| 소스 | 찾는 법 |
|------|---------|
| 로컬 미디어 / 비영어 미디어 | 주류 미디어에서 안 다룬 이야기가 있음 |
| LinkedIn 댓글/반응 | 본인이 직접 쓴 댓글에서 비공식 발언 발견 가능 |
| Substack Notes / Threads | 뉴스레터에는 안 쓰지만 짧게 올린 생각들 |
| 법적/재무 공개 자료 | 투자, 소송, 채용 공고 등에서 사실 확인 |
| 경쟁사/동료 발언 | 다른 사람이 이 사람에 대해 말한 것 |

**히든 스토리 등급:**

| 등급 | 설명 | 예시 |
|:----:|------|------|
| ★★★ | 본인이 한 번도 언급 안 한 것 | 외부 데이터에서만 발견 가능한 사실 |
| ★★☆ | 한 곳에서만 언급한 것 | 로컬 미디어/마이너 팟캐스트에서 한 번 나온 이야기 |
| ★☆☆ | 알려져 있지만 깊게 다뤄진 적 없는 것 | 사실은 공개됐지만 후속 질문이 없었던 것 |

## Phase 2: 주제 선정

### 2.1 Hot Take 추출

**목표:** 이 게스트가 EO 채널에 줄 수 있는 **고유한 가치**를 정의한다.

**Hot Take = 이 3가지가 동시에 충족되는 주장:**

```
1. 이 사람만 할 수 있는 이야기인가? (Uniqueness)
   → 경력, 경험, 포지션에서 나오는 것
   → 누구나 할 수 있는 일반론이면 탈락

2. 시청자가 반응할 수 있는가? (Reactivity)
   → 자기진단, 논쟁, 놀라움, 자격 도전 중 하나 이상 유발
   → 모두가 동의하는 안전한 말이면 탈락

3. 지금 시의성이 있는가? (Timeliness)
   → 6개월 전에도 할 수 있었던 말이면 탈락
   → 최근 변화, 최신 경험, 새 프레임워크에 기반해야 함
```

**Hot Take 평가 표:**

| Hot Take 후보 | Uniqueness | Reactivity | Timeliness | 총점 |
|--------------|:---:|:---:|:---:|:---:|
| (후보 1) | /5 | /5 | /5 | /15 |
| (후보 2) | /5 | /5 | /5 | /15 |

> 총점 12 이상이면 Lesson의 핵심 소재로 사용. 9 이하면 보조 소재 또는 탈락.

---

### 2.2 해묵은 얘기 필터링 (Staleness Filter)

**아래에 해당하면 무조건 탈락:**

| 필터 | 설명 | 예시 |
|------|------|------|
| **일반론** | 누구나 할 수 있는 말 | "AI가 세상을 바꿀 거예요" |
| **이미 소진** | 이 게스트가 3회 이상 한 이야기 | Phase 1.3에서 탐지 |
| **뻔한 결론** | 시청자가 클릭 전에 답을 알 수 있는 것 | "AI 시대에 적응해야 합니다" |
| **도구 나열** | 어떤 AI 도구 쓰나요 리스트 | "저는 ChatGPT와 Claude를 써요" |
| **과거 회고** | 이미 지나간 일에 대한 반복 | 2년 전 성공 스토리만 반복 |
| **포지션 토크** | 자기 회사/상품 홍보 | "우리 뉴스레터를 구독하세요" |

**Freshness 체크 질문:**

```
□ 이 이야기는 6개월 전에도 할 수 있었는가? → Yes면 재고
□ 이 이야기는 다른 게스트도 할 수 있는가? → Yes면 재고
□ 시청자가 제목만 보고 내용을 예측할 수 있는가? → Yes면 재고
□ 이 게스트의 최근 3개월 경험에 기반하는가? → No면 재고
```

---

### 2.3 Theme + Title/Thumbnail + Lessons 구조화

이것이 주제 선정의 **최종 산출물**이다. 이 구조가 확정되면 질문지 작성에 들어간다.

**포맷:**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

THEME (한 줄):
[이 인터뷰를 한 문장으로 정의]

TITLE CANDIDATES:
1. [제목 후보 A] (XX자)
2. [제목 후보 B] (XX자)
3. [제목 후보 C] (XX자)

THUMBNAIL CANDIDATES:
1. [썸네일 카피 A] — [전략: 자기진단/도발/대비 등]
2. [썸네일 카피 B] — [전략]

LESSON 1: [제목]
→ Core: [이 레슨의 핵심 한 줄]
→ Hot Take: [이 레슨에서 나올 가장 날카로운 발언]

LESSON 2: [제목]
→ Core: [핵심 한 줄]
→ Hot Take: [날카로운 발언]

LESSON 3: [제목]
→ Core: [핵심 한 줄]
→ Hot Take: [날카로운 발언]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**Lesson 구조 원칙:**

| 원칙 | 설명 |
|------|------|
| **3-Act 구조** | Lesson 1 = Hook (왜 들어야 하는가), Lesson 2 = Core (핵심 인사이트), Lesson 3 = Forward (그래서 뭘 해야 하는가) |
| **에스컬레이션** | 각 Lesson이 이전보다 더 날카롭거나 구체적이어야 함. 역순 금지 |
| **자기 완결** | 각 Lesson만 따로 클립으로 잘라도 의미가 있어야 함 |
| **게스트 고유성** | 모든 Lesson이 이 게스트의 경험/데이터에 기반해야 함. 일반론 금지 |

---

### 2.4 EO 채널 적합성 체크

```
□ EO Global 시청자(테크/빌더/글로벌 전문직)가 관심 가질 주제인가?
□ 영어로 진행 가능한가?
□ "이 영상 전에 없던 관점"이 있는가?
□ 논쟁 축이 최소 1개 존재하는가? (모두가 동의하면 콘텐츠가 약함)
□ 시청자가 자기 문제로 느낄 수 있는가? (자기투영 가능성)
□ Copy Workflow v5의 회피 패턴에 해당하지 않는가?
   - 전문 용어 중심 ✗
   - 타겟 한정 ✗
   - How-to 튜토리얼 톤 ✗
   - 소모된 YouTube 공식 ✗
```

## Phase 3: 질문지 작성

### 3.1 Lesson별 질문 설계

**질문 위계:**

| 마크 | 의미 | 기준 |
|:----:|------|------|
| 🔴 | Must ask — 절대 스킵 금지 | 이 질문이 빠지면 Lesson의 핵심이 빠짐 |
| ⚪ | Can skip — 시간 부족하면 생략 | 좋은 질문이지만 필수는 아님 |
| 🔵 | Series signature — 시리즈 고유 질문 | 매 에피소드 반복. 게스트마다 다른 답이 콘텐츠 |
| → | Follow-up — 답변이 얕을 때 | 본 질문 아래에 들여쓰기로 배치 |

**질문 작성 원칙:**

| 원칙 | 설명 | 예시 |
|------|------|------|
| **구체적 질문** | 넓은 질문 금지. 특정 경험/사건/수치를 겨냥 | ✗ "AI에 대해 어떻게 생각하세요?" ✓ "16개 앱을 만들면서 가장 놀란 순간은?" |
| **게스트 고유 질문** | 이 사람에게만 물을 수 있는 질문 | ✗ "AI 시대에 뭐가 중요한가요?" ✓ "30명의 AI 리더와 대화 후, 가장 의견이 갈린 주제는?" |
| **감정 유도 질문** | 사실이 아닌 느낌을 묻는 질문 포함 | ✗ "Reddit Talk이 중단됐죠?" ✓ "수백만이 쓰던 제품이 중단됐을 때 어떤 기분이었나요?" |
| **역질문** | 게스트의 주장에 도전하는 질문 | "그런데 99%가 Level 1이라고 하면, 당신의 프레임워크 자체가 대부분에게 의미 없는 거 아닌가요?" |
| **실화 요청** | 추상적 답변을 구체적 에피소드로 전환 | → "구체적인 예를 하나 들어줄 수 있나요? 이번 달에 있었던 일로." |

**Lesson당 질문 수 가이드:**

| 인터뷰 길이 | Lesson당 🔴 | Lesson당 ⚪ | 총 질문 |
|:----------:|:---:|:---:|:---:|
| 30분 | 2-3개 | 0-1개 | 8-10개 |
| 45분 | 3-4개 | 1개 | 12-15개 |
| 60분+ | 3-4개 | 1-2개 | 14-18개 |

---

### 3.2 Follow-up & 디플렉션 대응

모든 질문지에는 **디플렉션 패턴 표**를 포함한다.

**범용 디플렉션 패턴:**

| 패턴 | 게스트가 하는 행동 | 리디렉트 |
|------|-------------------|---------|
| 감정 → 숫자 | 느낌을 물었는데 수치로 답함 | "숫자는 알겠어요. 그때 **기분**은 어땠어요?" |
| 구체 → 일반 | 특정 사례를 물었는데 업계 트렌드로 답함 | "업계 전체 말고, **당신 팀**에서는 어땠나요?" |
| 현재 → 홍보 | 답변 중 자기 상품/서비스 홍보로 전환 | "그건 알겠고, 그 전에 — (원래 질문으로 복귀)" |
| 깊이 → 표면 | 민감한 질문에 피상적으로 답함 | "그건 공식 답변 같아요. 비공식적으로는?" |
| 새 질문 → 구 답변 | 새로운 질문에 이전 인터뷰에서 한 답변을 반복 | "그 이야기는 알아요. 그 **이후에** 뭐가 달라졌나요?" |

> Phase 1.3에서 탐지한 게스트 고유 디플렉션 패턴을 여기에 추가한다.

---

### 3.3 팩트체크 레퍼런스

모든 질문지에 아래 3개 카테고리를 포함:

**Verified (안전하게 인용 가능)**
- 복수 소스에서 확인된 사실
- 공개 발표, 공식 프로필

**Unverified (인터뷰 중 확인 필요 또는 인용 금지)**
- 단일 소스에서만 나온 정보
- 수치가 소스마다 다른 것
- "~에 따르면" 형태로만 사용 가능

**Already Exhausted (다시 묻지 말 것)**
- Phase 1.3에서 탐지된 반복 질문/답변
- 이미 널리 알려진 기본 정보

---

### 3.4 프로덕션 노트

모든 질문지 마지막에 포함:

```
■ 언어: 영어 / 한국어
■ 예상 러닝타임: XX분
■ 게스트 커뮤니케이션 스타일: (감정적/분석적/스토리텔링/방어적 등)
■ 인터뷰 전략: (신뢰 먼저 쌓기/도발로 시작 등)
■ 시리즈 고유 질문: (있으면)
■ 비주얼 기회: (프로토타입 데모, 차트, 제품 시연 등)

■ TIMELINE SUMMARY:
| Time | Lesson | Must-Ask | Skippable | Goal |
|------|--------|----------|-----------|------|
| 0-XX | Lesson 1 | QX, QX | QX | (목표) |
| ...  | ...     | ...      | ...       | ... |

■ PACING RULE:
- 질문이 터지면 (게스트가 깊이 들어가면) → 그 자리에 머물기. 다음 ⚪ 스킵.
- 질문이 안 먹히면 → → follow-up 한 번 시도 후, 안 되면 다음 🔴로 이동.
```

## 시리즈별 커스텀 요소

각 시리즈는 고유한 요소를 가질 수 있다. 아래 섹션에 시리즈별 추가 규칙을 기록한다.

### Thinking Mode

```
타겟 시청자: 빌더 (PM, 엔지니어, 디자이너, 파운더 — 특정 직군 한정 금지)
톤: 실용적이고 날카로움. "그래서 나는 뭘 해야 하지?"에 답하는 인터뷰.
핵심 원칙:
- 이론보다 실전 (이 사람이 직접 한 경험 위주)
- 해묵은 AI 얘기 금지 (Phase 2.2 필터 엄격 적용)
- 빌더의 행동을 바꿀 수 있는 인사이트 우선
시리즈 고유 질문: (아직 미정 — 에피소드 축적되면 추가)
```

### Before 25

```
타겟 시청자: 젊은 빌더/기업가
시리즈 고유 질문 (🔵):
- Opening: "Before you turned 25, what was the 'this won't work' you heard the most?"
- Midpoint: "What are friends your age doing right now?"
- Closing: "Ten years from now, if you look back at your 25-year-old self, what would you say?"
```

## 품질 체크리스트

질문지 최종 제출 전 확인:

```
■ Phase 1 (리서치)
□ 프로필이 완전한가? (경력 타임라인, 링크, 유명도)
□ 기존 인터뷰 최소 3개 이상 분석했는가?

□ 콘텐츠 감사로 최근 관심사/프레임워크를 파악했는가?

■ Phase 2 (주제)
□ Hot Take가 최소 3개 이상 추출됐는가?
□ 해묵은 얘기 필터를 통과했는가?
□ Theme + Title/Thumbnail + Lessons 구조가 확정됐는가?
□ EO 채널 적합성 체크를 통과했는가?

■ Phase 3 (질문지)
□ 모든 Lesson에 🔴 질문이 최소 2개 이상 있는가?
□ 모든 🔴 질문에 → follow-up이 최소 1개 있는가?
□ 디플렉션 대응 표가 있는가?
□ 팩트체크 3개 카테고리가 분류됐는가?
□ 프로덕션 노트와 타임라인이 있는가?
□ 파일이 interview_prep_{guest}_{date}.md 형식으로 저장됐는가?
```
