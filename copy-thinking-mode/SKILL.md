# Copy Workflow — The Thinking Mode Series

EO Global "The Thinking Mode" 시리즈(교수·학자 인터뷰)를 위한 제목·썸네일·인트로 최적화 스킬.
기반: EO Copy Workflow v6 (355-video dataset + A/B 3회 + 댓글 397개 분석)

## 입력

사용자가 아래 정보를 제공한다. 형태는 자유롭다:

```
■ 게스트 정보
이름:
회사/소속:
현재 직함:
유명도: [대중적 유명 / 업계 유명 / 무명]

■ 스크립트 (전문 또는 요약)

■ 이전 영상 댓글 (선택)

■ 추가 요청 (선택)
예: "감정형 위주", "일반 대중 타겟", "글로벌 앵글 강조" 등
```

## 워크플로우

사용자 입력이 들어오면, 아래 **4개 모드** 중 적합한 것을 자동 판단하거나 사용자에게 묻는다.

### Mode A: 풀 파이프라인 (Step 1 → 1.5 → 2 → 3 → 3.5 → 4)

스크립트 + 게스트 정보가 있을 때 실행. 가장 일반적인 모드.

### Mode B: 콘셉트 Only (Step 3 → 3.5 → 4)

사용자가 이미 훅/트렌드 분석을 마치고 콘셉트만 필요할 때.

### Mode C: A/B 결과 분석 (Step 6)

A/B 테스트 결과 데이터가 입력될 때. WTS 기반 승패 분석 + 학습 메모.

### Mode D: 분기 업데이트 (Step 7)

3개월치 YouTube Studio CSV + A/B 결과 모음이 입력될 때. 패턴 풀 업데이트.

## Step 1: Hook Inventory (스크립트 분석)

스크립트에서 아래 7종 Hook을 추출한다:

| Hook 유형 | 설명 | 예시 |
|-----------|------|------|
| **신뢰 Hook** | 검증 가능한 성과, 투자, 수치 | a16z+Sequoia 공동투자, Stanford's first AI class |
| **드라마 Hook** | 희생, 갈등, 반전, 언더독 | 1000 applications, 2 callbacks |
| **숫자 Hook** | 달러, 유저수, 나이, 퍼센트 | Top 1%, 0.1%, 3 Forces |
| **반직관 Hook** | 통념을 뒤집는 사실 | More agents = worse code |
| **최상급 Hook** | 최초, 최대, 유일, 가장 | Stanford's first AI-native software dev class |
| **변환 Hook** | 역할/정체성의 변화 | From writing code to managing agents |
| **지리/글로벌 Hook** | 실리콘밸리 밖의 실제 사례 | Kenya, Kazakhstan, India — 스크립트에 나온 국가만 |

**게스트 유명도별 제목 전략:**

| 유명도 | 제목 전략 | 파이프(\|) 뒤 |
|:---:|-----------|-----------|
| 대중적 유명 (일론 머스크급) | 이름이 곧 훅 | 이름만 |
| 업계 유명 (Sam Altman급) | 이름 + 직함/회사 | 이름, 회사 |
| 니치/무명 | 이름 쓰지 않음. 스토리/숫자로 훅 | 이름, 회사 (태그만) |

**Hook 출처 검증 (필수):**
- 모든 숫자/주장을 스크립트에서 직접 확인
- 게스트가 직접 말한 것 vs AI가 추론한 것 구분
- 30초 미만 분량 소재는 메인 콘셉트 사용 금지

## Step 1.5: Comment Intelligence (댓글 분석)

댓글이 제공되면 아래를 수행:

### 댓글 유형 분류 (5 Types)

| 유형 | Copy 활용도 |
|------|:-----------:|
| **Quote Extraction** — 영상 속 명언 공유 | ★★★ |
| **Challenge / Skeptic** — 자격 의심, 내용 반박 | ★★★★★ |
| **Self-Projection** — "나도 ~", "나는 ~인데 공감" | ★★★★ |
| **Discovery Joy** — "알고리즘 감사" | ★★★ |
| **Debate Trigger** — 찬반 격렬한 싸움 | ★★★★★ |

### 논쟁 축(Debate Axis) 도출

답글 수 기준 상위 5개 댓글 → 논쟁 축 추출:

| 논쟁 축 | 트리거 |
|---------|--------|
| **자격 도전 (Credential Threat)** | 특정 집단의 정체성을 건드림 |
| **시스템 비판 (System Critique)** | 숫자의 불균형이 불공정 감각 유발 |
| **인간 본성 논쟁 (Human Nature)** | AI의 미래에 대한 철학적 질문 |
| **과대평가 회의 (Hype Skepticism)** | 과장된 제목이 오히려 클릭 유발 |
| **사실 검증 (Fact Check)** | 극단적 수치/주장이 검증 욕구 유발 |

### 논쟁도 점수

| 점수 | 기준 |
|:----:|------|
| ★★★★★ | 상위 댓글 50%+ 가 Challenge/Debate |
| ★★★★☆ | 3개 이상 논쟁 댓글 (답글 3+) |
| ★★★☆☆ | 논쟁 1-2개, 대부분 긍정 |
| ★★☆☆☆ | 거의 모든 댓글이 긍정/감사 — **카피가 너무 안전** |
| ★☆☆☆☆ | 댓글 자체가 적음 |

### 시청자 생성 표현(Viewer-Generated Language) 추출

댓글에서 직접 만든 표현을 추출하여 다음 에피소드 카피에 활용.

> **핵심 원칙**: 논쟁을 유발하는 제목이 알고리즘도 밀어줌. 모두가 동의하는 제목은 클릭 유도력이 약함.

## Step 2: 트렌드 인텔리전스 (5-Layer Scan)

WebSearch를 활용하여 아래 5개 레이어를 스캔:

| Layer | 설명 |
|-------|------|
| **1. 업계 버즈** | 게스트 산업 최근 1-2주 뉴스, 검색량 추이 |
| **2. YouTube 검색 수요** | 실제 검색 키워드 (vidIQ/TubeBuddy 기준) |
| **3. 경쟁 채널 갭** | YC/DOAC/Lenny's가 안 다루는 주제 |
| **4. 경쟁 채널 카피 패턴** | DOAC, Lex, Impact Theory, Ali Abdaal, Colin & Samir |
| **5. 실시간 트렌드 매칭** | 게시 직전 1-3일 빅뉴스와 겹치는 키워드 |

**EO 적용 원칙:**
- 구체적 사실/숫자 + 키워드 조합이 EO에 적합 (Impact Theory/Lex 스타일)
- 모호한 감정 트리거는 EO 규모에서 효과 약함 (DOAC 스타일 불가)
- 트렌드 키워드는 영상 내용과 실제 연결될 때만 사용

## Step 3: 콘셉트 개발

제목과 썸네일을 **동시에 하나의 콘셉트 카드**로 설계. **총 10개** 생성:

| 카테고리 | 개수 |
|---------|:---:|
| 검증 콘셉트 (성과 검증된 패턴) | 3-4개 |
| 최적화 콘셉트 (데이터상 최고 조합) | 3-4개 |
| 실험 콘셉트 (새 앵글, 실패 가능) | 2-3개 |

### 썸네일 카피 11원칙

1. 제목과 **다른 레이어** (요약 금지)
2. **통념 뒤집기** or **인사이트 암시**
3. **시청자 참여** (질문, 도발, 자기 투영)
4. **구체성 필수** — 모호한 감정 태그 절대 금지
5. **영상 주제 키워드 포함**
6. **영상 전체 내용과 일치** — 30초 소재 확대 금지
7. **자기투영 가능해야 함** — "나한테 해당되나?"
8. **정보 밀도 높을수록 좋다** — 텍스트 + 그래프 + 태그가 서로 보강
9. **타겟 범위를 넓히는 단어 선택** — "Junior Dev" ✗ → "Software Engineer" ✓
10. **새로운 정체성/프레임워크 제안 > 실수 지적** — aspirational > guilt
11. **1초 모바일 가독성 테스트** — 모바일에서 엄지 스크롤 중 1초 안에 의미 전달

### 썸네일 비주얼 디렉션 (6종)

| 비주얼 유형 | 강도 | 사용 조건 |
|------------|:---:|----------|
| **K-그래프 (분기 그래프)** | ★★★★★ | "격차" "상위 vs 하위" 주제 |
| **변환 화살표 (A → B)** | ★★★★☆ | "From X to Y" 변환형 콘셉트 |
| **글로브 + 국가명** | ★★★★☆ | 글로벌 앵글. 사진 대신 텍스트(국가명) |
| **대비 레이아웃** | ★★★★☆ | "RURAL / URBAN" 등 검증 패턴 |
| **숫자 강조** | ★★★☆☆ | $100M, 50 PEOPLE 등 충격적 숫자 |
| **인물 비유** | ★★★☆☆ | 유명 인물 이미지/이름으로 비유 |

### 썸네일 카피 금지사항

- 모호한 감정 태그 ("YOUR JOB IS NEXT")
- 스테레오타입 비주얼 (개발도상국 = 빈곤)
- 맥락 없는 짧은 문구 ("NOT FOR DANCING")
- 타겟 한정 라벨 ("Junior Dev:", "For Beginners")
- 업계 전문 용어 ("Agentic Engineering 101")
- 소모된 YouTube 공식 ("The Mistake X% Make")
- 실수 지적형 ("More Agents ≠ Better Code")
- 철지난 트렌드 ("STOP USING CHATGPT")
- HR/업계 전문 톤 ("How to Find the Best Talent")

### 제목 작성 원칙

- 썸네일과 같은 영상처럼 읽혀야 함
- em dash(—) 사용 금지 → 콤마/마침표
- 약어 금지: "CMU" ✗ → "Carnegie Mellon" ✓
- 현재 직함만 사용
- 한 문장 원칙
- **100자 제한** (파이프+게스트명 포함, 80자 이내 이상적)
- **"From X to Y" 변환형 우선** (38% vs "How to" 29.9%)
- **직접 도발 > 중립 관찰** ("Aren't Ready" > "Do Differently")
- 파이프 뒤 기관명 — 썸네일에 있으면 제목에서 생략 가능

### 스테레오타입 회피 원칙

글로벌/개발도상국 앵글 시:
- **국가 이름 텍스트** — 글로브 위에 게스트가 직접 언급한 국가명만 배치
- 빈곤/먼지길 이미지 절대 금지 — EO의 aspirational 톤과 불일치
- 국가 > 도시 (게스트가 말한 단위 그대로)

## Step 3.5: 인트로 설계 + Three-Combo 정렬

### Three-Combo 원칙

```
썸네일 → "무엇에 관한 영상인가?" (1초 시각 약속)
    ↕ 같은 방향
제목   → "왜 이 영상을 봐야 하는가?" (구체적 설명/도발)
    ↕ 같은 방향
인트로 → "그 약속을 60초 안에 증명" (데이터/스토리로 증거)
```

### 인트로 설계 6원칙

1. **60초 이내**
2. **두 축 구조: Fear Open → Aspiration Land**
3. **구체적 숫자 1개 이상**
4. **게스트의 직접 발언 활용** (학술 게스트에게 특히 효과적)
5. **썸네일 약속을 15초 안에 터치**
6. **인트로 끝 = 영상의 핵심 프레임 선언**

### 인트로 구조 템플릿

| 파트 | 시간 | 역할 |
|------|:---:|------|
| **HOOK** | 0-10초 | 격차/충격 선언 |
| **EVIDENCE** | 10-35초 | 데이터/스토리로 증명 |
| **TWIST** | 35-50초 | 추가 반전/심화 |
| **FRAME** | 50-60초 | 핵심 프레임 제시 (aspiration) |

## 회피 패턴

- 업계 전문 용어 + 101/가이드 톤 (WTS 29.9% 최하위)
- 타겟 한정 라벨 ("Junior Dev" → 시니어 이탈)
- "How to" 튜토리얼 프레임 ("From X to Y"에 완패)
- 소모된 YouTube 공식 ("The Mistake X% Make")

### Three-Combo 정렬 체크 (콘셉트별)

```
□ 썸네일과 제목이 같은 영상처럼 읽히는가?
□ 썸네일과 제목이 서로 다른 레이어인가?
□ 썸네일만 보고도 클릭 이유가 있는가?
□ 제목만 읽어도 영상 방향이 잡히는가?
□ 영상 전체 내용의 핵심 주제와 일치하는가?
□ 시청자가 자기 문제로 느낄 수 있는가?
□ 이전 영상과 차별화되는가?
□ 타겟이 특정 레벨로 한정되지 않는가?
□ 썸네일 정보 밀도가 충분한가?
□ 썸네일 1초 모바일 가독성 통과?
□ 비주얼 디렉션이 구체적인가?
□ 스크립트에 없는 숫자/주장 사용하지 않는가?
□ 인트로가 썸네일 약속을 15초 안에 터치하는가?
```

### A/B 테스트 3세트 추천

10개 중 Top 3를 선정. **반드시 서로 다른 프레임**:

| 세트 | 역할 |
|------|------|
| A | 검증 패턴 (가장 안전) |
| B | 새로운 프레임/비유 (중간 리스크) |
| C | 실험 앵글 (고위험 고수익) |

### 인트로 구조 제안

Top 3 세트에 호환되는 인트로를 HOOK-EVIDENCE-TWIST-FRAME 구조로 제안.

## 규칙

- 무명 기업/게스트 이름은 제목 메인에 넣지 않는다. 파이프(|) 뒤에만
- 전문용어/업계 용어/약어 제목에 넣지 않는다
- 숫자/주장은 반드시 스크립트 원문 확인
- 논쟁을 유발하는 카피 방향 최소 2개 포함
- 각 콘셉트에 CTR 예측 점수 산출
- 3세트는 각각 다른 프레임으로 구성
- 출력 언어: 제목/썸네일 카피는 **영어**, 설명/분석은 **한국어**
