# Copy Workflow — The Thinking Mode Series

EO Global "The Thinking Mode" 시리즈(교수·학자 인터뷰)를 위한 제목·썸네일·인트로 최적화 스킬.

## 마스터 워크플로우 참조

**실행 전 반드시 아래 파일을 Read:**

```
~/.claude/skills/references/eo_copy_workflow.md
```

이 파일에 모든 세부 원칙, CTR 스코어링 모델, 검증 패턴, 회피 패턴, 썸네일 변형 시스템, 조합 세트 포맷이 포함되어 있다. SKILL.md는 모드 분기와 입출력 포맷만 정의하고, **원칙/스코어링/패턴은 마스터 문서를 따른다.**

---

## 입력

사용자가 아래 정보를 제공한다. 형태는 자유롭다:

```
■ 게스트 정보
이름:
회사/소속:
현재 직함:
유명도: [대중적 유명 / 업계 유명 / 무명]

■ 스크립트 (전문 또는 요약)

■ 이전 영상 댓글 (선택)

■ 추가 요청 (선택)
예: "감정형 위주", "일반 대중 타겟", "글로벌 앵글 강조" 등
```

---

## 모드 분기

사용자 입력이 들어오면, 아래 **4개 모드** 중 적합한 것을 자동 판단하거나 사용자에게 묻는다.

### Mode A: 풀 파이프라인 (Step 1 → 1.5 → 2 → 3 → 3.5 → 4)

스크립트 + 게스트 정보가 있을 때 실행. 가장 일반적인 모드.

### Mode B: 콘셉트 Only (Step 3 → 3.5 → 4)

사용자가 이미 훅/트렌드 분석을 마치고 콘셉트만 필요할 때.

### Mode C: A/B 결과 분석 (Step 6)

A/B 테스트 결과 데이터가 입력될 때. WTS 기반 승패 분석 + 학습 메모.

### Mode D: 분기 업데이트 (Step 7)

3개월치 YouTube Studio CSV + A/B 결과 모음이 입력될 때. 패턴 풀 업데이트.

## 실행 절차 (Mode A 기준)

### Step 1: Hook Inventory
- 마스터 문서의 7종 Hook 유형에 따라 스크립트에서 추출
- 게스트 유명도 판단 → 제목 전략 결정
- **Hook 출처 검증 필수**: 숫자/주장 스크립트 원문 확인

### Step 1.5: Comment Intelligence
- 댓글이 제공되면: 5 Types 분류, 논쟁 축 도출, 논쟁도 점수, 시청자 생성 표현 추출

### Step 2: 트렌드 인텔리전스
- WebSearch로 5-Layer Scan 실행 (마스터 문서 참조)
- 경쟁 채널 갭 분석 포함

### Step 3: 콘셉트 개발
- **카피 3대 우선순위: 구체성 > 시의성 > 궁금증 유발** (마스터 문서 참조)
- 총 **15개** 콘셉트 (검증 4-5 / 최적화 4-5 / 실험 4-5)
- 썸네일 카피 11원칙 + 비주얼 디렉션 6종 + **썸네일 변형 시스템 3타입** (마스터 문서 참조)
- 제목 작성 원칙 + 금지사항 (마스터 문서 참조)

### Step 3.5: 인트로 설계 + Three-Combo 정렬
- **인트로 범위: 영상 시작 ~ 게스트 자기소개 직전**
- 인트로 설계 6원칙 + HOOK-EVIDENCE-TWIST-FRAME 구조 (마스터 문서 참조)
- Three-Combo 정렬 체크

### Step 4: CTR 예측 + 근거 매칭
- CTR Scoring Model (마스터 문서의 가산 19개 + 감산 10개 요소 적용)

## 최종 출력

### 1. 15개 콘셉트 카드

마스터 문서의 스코어링 출력 형식에 따라 출력.

### 2. Title+Thumbnail 조합 세트

Top 콘셉트 × 변형(텍스트 중심/하이브리드/비주얼 중심) = 세트 단위 출력.
마스터 문서의 조합 세트 출력 포맷 참조.

### 3. Three-Combo 정렬 체크 (콘셉트별)

```
□ 썸네일과 제목이 같은 영상처럼 읽히는가?
□ 썸네일과 제목이 서로 다른 레이어인가?
□ 썸네일만 보고도 클릭 이유가 있는가?
□ 제목만 읽어도 영상 방향이 잡히는가?
□ 영상 전체 내용의 핵심 주제와 일치하는가?
□ 시청자가 자기 문제로 느낄 수 있는가?
□ 이전 영상과 차별화되는가?
□ 타겟이 특정 레벨로 한정되지 않는가?
□ 썸네일 정보 밀도가 충분한가?
□ 썸네일 1초 모바일 가독성 통과?
□ 비주얼 디렉션이 구체적인가?
□ 스크립트에 없는 숫자/주장 사용하지 않는가?
□ 인트로가 썸네일 약속을 15초 안에 터치하는가?
```

### 4. A/B 테스트 3세트 추천

15개 중 Top 3를 선정. **반드시 서로 다른 프레임**:

| 세트 | 역할 |
|------|------|
| A | 검증 패턴 (가장 안전) |
| B | 새로운 프레임/비유 (중간 리스크) |
| C | 실험 앵글 (고위험 고수익) |

### 5. 인트로 구조 제안

Top 3 세트에 호환되는 인트로를 HOOK-EVIDENCE-TWIST-FRAME 구조로 제안.

## 규칙

- 무명 기업/게스트 이름은 제목 메인에 넣지 않는다. 파이프(|) 뒤에만
- 전문용어/업계 용어/약어 제목에 넣지 않는다
- 숫자/주장은 반드시 스크립트 원문 확인
- 논쟁을 유발하는 카피 방향 최소 2개 포함
- 각 콘셉트에 CTR 예측 점수 산출
- 3세트는 각각 다른 프레임으로 구성
- 출력 언어: 제목/썸네일 카피는 **영어**, 설명/분석은 **한국어**
- **문서 변경 시 반드시 사용자에게 확인 후 진행**
