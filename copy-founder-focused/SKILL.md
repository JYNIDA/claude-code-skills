# Copy Workflow — Founder Focused Series

EO Global "Founder Focused" 시리즈(창업가·CEO 인터뷰)를 위한 제목·썸네일·인트로 최적화 스킬.
기반: EO Copy Workflow v6 (355-video dataset + A/B 3회 + 댓글 397개 분석) + Founder 전용 학습 (Pensive AI CEO 세션 등)

**copy-thinking-mode와의 핵심 차이:**
- Drama vs Strategy 자동 감지 (Step 0)
- Founder Hook 9종 (학자용 7종 대체)
- Timeliness Check 필수 (Step 2.5)
- Self-Projection 5유형 필수 태그
- 팩트체크 프로토콜 강화 (펀딩/학력/경쟁사)
- 썸네일 비주얼 디렉션 7종 (전면 교체)
- A/B 3세트 = Drama / Strategy / Experimental 고정
- YC-Educational 프레이밍 옵션 (Strategy 콘셉트에 적용 가능)

**포지셔닝 참고 — EO vs YC vs DOAC:**
| 채널 | 스타일 | 강점 |
|------|--------|------|
| **YC** | 교육형. 짧고 밀도 높은 프레임워크 전달. "Startup School" | 실용성, 행동 가능성 |
| **DOAC** | 감정형. 긴 인터뷰, 개인 서사 중심 | 감정 몰입, 바이럴 |
| **EO Global** | **서사 × 교육 하이브리드**. 인터뷰 기반 서사 + 구조화된 교훈 | 차별화된 깊이 |

EO의 고유 포지션: YC의 교육적 밀도 + DOAC의 인터뷰 서사력. 파운더의 경험에서 **추출한 프레임워크**를 시청자가 가져갈 수 있도록 설계한다.

## 입력

사용자가 아래 정보를 제공한다. 형태는 자유롭다:

```
■ 게스트 정보
이름:
회사/소속:
현재 직함:
유명도: [대중적 유명 / 업계 유명 / 무명]

■ 스크립트 (전문 또는 요약)

■ 이전 영상 댓글 (선택)

■ 추가 요청 (선택)
예: "드라마형 위주", "전략형 강조", "글로벌 앵글 강조" 등
```

## 워크플로우

사용자 입력이 들어오면, 아래 **4개 모드** 중 적합한 것을 자동 판단하거나 사용자에게 묻는다.

### Mode A: 풀 파이프라인 (Step 0 → 1 → 1.5 → 2 → 2.5 → 3 → 3.5 → 4)

스크립트 + 게스트 정보가 있을 때 실행. 가장 일반적인 모드.

### Mode B: 콘셉트 Only (Step 3 → 3.5 → 4)

사용자가 이미 훅/트렌드 분석을 마치고 콘셉트만 필요할 때.

### Mode C: A/B 결과 분석 (Step 6)

A/B 테스트 결과 데이터가 입력될 때. WTS 기반 승패 분석 + 학습 메모.

### Mode D: 분기 업데이트 (Step 7)

3개월치 YouTube Studio CSV + A/B 결과 모음이 입력될 때. 패턴 풀 업데이트.

## Step 0: Drama vs Strategy 자동 감지 (NEW)

스크립트를 분석하여 이 파운더가 **드라마형**인지 **전략형**인지 판정한다. 이 결과가 Step 3 콘셉트 배분을 결정한다.

### 8개 신호 분석

| # | 신호 | Drama 지표 | Strategy 지표 |
|:-:|------|-----------|--------------|
| 1 | **오리진 스토리 감정 강도** | 역경·고난·감정 표현 풍부 (3+ 감정 피크) | 담백하게 서술, 논리적 흐름 |
| 2 | **피봇 횟수** | 2회 이상 방향 전환 | 0-1회, 일관된 방향 |
| 3 | **실패 서사 비중** | 스크립트 20%+ 가 실패/위기 이야기 | 10% 미만 |
| 4 | **방법론/프레임워크 분량** | 10% 미만 | 스크립트 25%+ 가 방법론·프레임워크 |
| 5 | **숫자 유형** | 감정 대비 숫자 ("$0 → $10M", "1000 rejections") | 분석 숫자 ("CAC $12", "NPS 72", "3-step framework") |
| 6 | **게스트 발언 톤** | 개인적, 고백적, 1인칭 서사 | 분석적, 교훈적, 프레임워크 제시 |
| 7 | **포기/희생 언급** | 구체적 희생 스토리 존재 (직장, 관계, 건강 등) | 거의 없음 |
| 8 | **청중 투영 가능성** | "나도 저 상황이면..." 반응 유발 | "나도 저 방법을 쓸 수 있겠다" 반응 유발 |

### 판정 기준

| 판정 | 조건 | Step 3 콘셉트 배분 |
|------|------|-------------------|
| **Drama 우세** | Drama 신호 5개 이상 | Drama 6 : Strategy 3 : Experimental 1 |
| **Strategy 우세** | Strategy 신호 5개 이상 | Strategy 6 : Drama 3 : Experimental 1 |
| **하이브리드** | 4:4 또는 차이 1개 이내 | Drama 4 : Strategy 4 : Experimental 2 |

### 출력 형식

```
━━━ Step 0: Drama vs Strategy 판정 ━━━━━━━━━━━━━━━
판정: [Drama 우세 / Strategy 우세 / 하이브리드]
신호 분석:
  Drama: X/8 — [해당 신호 나열]
  Strategy: X/8 — [해당 신호 나열]
콘셉트 배분: Drama X : Strategy X : Experimental X
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Step 1: Hook Inventory (스크립트 분석)

스크립트에서 아래 **Founder Hook 9종**을 추출한다:

| Hook 유형 | 설명 | 예시 |
|-----------|------|------|
| **Origin Story** | 창업 시작점, 결정적 순간 | "Quit Goldman Sachs at 25 to build from a garage" |
| **Failure/Pivot** | 실패 횟수, 방향 전환 | "3 failed startups before finding product-market fit" |
| **Traction Numbers** | 매출/유저/펀딩 극적 대비 | "$0 to $50M ARR in 18 months" |
| **Founder Philosophy** | 의사결정 프레임워크, 멘탈 모델 | "I fire myself from a role every 6 months" |
| **Underdog/Against-Odds** | 나이/배경/자격 갭 | "No CS degree, built a $1B company" |
| **Timeliness** | 왜 이 파운더가 지금 중요한가 | "Just raised Series C amid AI gold rush" |
| **Industry Disruption** | 반직관적 시장 베팅 | "Bet against SaaS when everyone was building SaaS" |
| **Sacrifice/Cost** | 창업 위해 포기한 것 | "Sold my house, maxed 3 credit cards" |
| **Contrarian Bet** | 모두가 틀렸다고 한 방향에 베팅 | "Everyone said enterprise AI was dead. We went all in." |
| **Founder Lesson** | 파운더 경험에서 추출한 구조화된 교훈 (YC-Educational 스타일) | "The 3 things I wish I knew before raising Series A" |

> **Founder Lesson Hook 활용 가이드 (YC-Educational 프레이밍):**
> - YC 채널처럼 실용적·구조화된 교훈을 제목/썸네일에 반영
> - 단, EO는 순수 교육 채널이 아니므로 **서사와 결합**해야 함
> - Good: "He Built a $100M Company Using 3 Rules" (서사 + 프레임워크)
> - Bad: "3 Rules for Building a Startup" (순수 교육 — YC 영역)
> - 핵심: 파운더의 **개인 경험에서 추출**한 교훈이어야 함 (범용 조언 ✗)

**게스트 유명도별 제목 전략:**

| 유명도 | 제목 전략 | 파이프(\|) 뒤 |
|:---:|-----------|-----------|
| 대중적 유명 (일론 머스크급) | 이름이 곧 훅 | 이름만 |
| 업계 유명 (Sam Altman급) | 이름 + 직함/회사 | 이름, 회사 |
| 니치/무명 | 이름 쓰지 않음. 스토리/숫자로 훅 | 이름, 회사 (태그만) |

**Hook 출처 검증 (필수):**
- 모든 숫자/주장을 스크립트에서 직접 확인
- 게스트가 직접 말한 것 vs AI가 추론한 것 구분
- 30초 미만 분량 소재는 메인 콘셉트 사용 금지

## Step 1.5: Comment Intelligence (댓글 분석)

댓글이 제공되면 아래를 수행:

### 댓글 유형 분류 (5 Types)

| 유형 | Copy 활용도 |
|------|:-----------:|
| **Quote Extraction** — 영상 속 명언 공유 | ★★★ |
| **Challenge / Skeptic** — 자격 의심, 내용 반박 | ★★★★★ |
| **Self-Projection** — "나도 ~", "나는 ~인데 공감" | ★★★★ |
| **Discovery Joy** — "알고리즘 감사" | ★★★ |
| **Debate Trigger** — 찬반 격렬한 싸움 | ★★★★★ |

### 논쟁 축(Debate Axis) 도출

답글 수 기준 상위 5개 댓글 → 논쟁 축 추출:

| 논쟁 축 | 트리거 |
|---------|--------|
| **자격 도전 (Credential Threat)** | 특정 집단의 정체성을 건드림 |
| **시스템 비판 (System Critique)** | 숫자의 불균형이 불공정 감각 유발 |
| **인간 본성 논쟁 (Human Nature)** | 창업/성공에 대한 철학적 질문 |
| **과대평가 회의 (Hype Skepticism)** | 과장된 제목이 오히려 클릭 유발 |
| **사실 검증 (Fact Check)** | 극단적 수치/주장이 검증 욕구 유발 |
| **생존자 편향 (Survivorship Bias)** | "운이 좋았을 뿐" — 성공 스토리에 대한 회의 |

### 논쟁도 점수

| 점수 | 기준 |
|:----:|------|
| ★★★★★ | 상위 댓글 50%+ 가 Challenge/Debate |
| ★★★★☆ | 3개 이상 논쟁 댓글 (답글 3+) |
| ★★★☆☆ | 논쟁 1-2개, 대부분 긍정 |
| ★★☆☆☆ | 거의 모든 댓글이 긍정/감사 — **카피가 너무 안전** |
| ★☆☆☆☆ | 댓글 자체가 적음 |

### 시청자 생성 표현(Viewer-Generated Language) 추출

댓글에서 직접 만든 표현을 추출하여 다음 에피소드 카피에 활용.

> **핵심 원칙**: 논쟁을 유발하는 제목이 알고리즘도 밀어줌. 모두가 동의하는 제목은 클릭 유도력이 약함.

## Step 2: 트렌드 인텔리전스 (5-Layer Scan)

WebSearch를 활용하여 아래 5개 레이어를 스캔:

| Layer | 설명 |
|-------|------|
| **1. 업계 버즈** | 게스트 산업 최근 1-2주 뉴스, 검색량 추이 |
| **2. YouTube 검색 수요** | 실제 검색 키워드 (vidIQ/TubeBuddy 기준) |
| **3. 경쟁 채널 갭** | YC/DOAC/Lenny's가 안 다루는 주제 |
| **4. 경쟁 채널 카피 패턴** | DOAC, Lex, Impact Theory, Ali Abdaal, Colin & Samir |
| **5. 실시간 트렌드 매칭** | 게시 직전 1-3일 빅뉴스와 겹치는 키워드 |

**EO 적용 원칙:**
- 구체적 사실/숫자 + 키워드 조합이 EO에 적합 (Impact Theory/Lex 스타일)
- 모호한 감정 트리거는 EO 규모에서 효과 약함 (DOAC 스타일 불가)
- 트렌드 키워드는 영상 내용과 실제 연결될 때만 사용

## Step 2.5: Timeliness Check (필수)

**"왜 이 파운더를 지금 봐야 하는가?"** — 4층위 검증.

이 단계는 건너뛸 수 없다. Timeliness가 약하면 CTR에 직접 감산된다.

### 4-Layer Timeliness 검증

| 층위 | 설명 | 강도 | 예시 |
|------|------|:---:|------|
| **1. Breaking News** | 게스트/회사가 최근 1-2주 내 뉴스에 등장 | ★★★★★ | IPO 발표, 대형 인수, 해고 논란 |
| **2. Industry Moment** | 게스트 산업 전체에 큰 변곡점 | ★★★★☆ | AI 규제법 통과, 신기술 출시 |
| **3. Cultural Moment** | 대중 문화/사회적 흐름과 맞닿음 | ★★★☆☆ | 대퇴직 트렌드, 사이드허슬 열풍 |
| **4. Evergreen Trigger** | 시의성은 약하지만 반복 검색되는 주제 | ★★☆☆☆ | "how to start a startup", "quit your job" |

### 판정 및 플래그

| 판정 | 조건 | 액션 |
|------|------|------|
| **Strong** | Layer 1 또는 Layer 2 해당 | 해당 뉴스/이벤트를 제목·썸네일에 반영 |
| **Moderate** | Layer 3 해당 | Cultural Moment 키워드를 콘셉트 2-3개에 반영 |
| **Weak** | Layer 4만 해당 | ⚠️ **경고**: Timeliness 부재. CTR -0.20% 감산 적용. 대안: 숫자/스토리 훅 강화 |
| **None** | 해당 없음 | ⚠️⚠️ **강력 경고**: 게시 타이밍 재고 권장. 숫자/스토리 훅으로 보완 필수 |

### 출력 형식

```
━━━ Step 2.5: Timeliness Check ━━━━━━━━━━━━━━━━━━
판정: [Strong / Moderate / Weak / None]
Layer:
  1. Breaking News: [해당 사항 또는 "없음"]
  2. Industry Moment: [해당 사항 또는 "없음"]
  3. Cultural Moment: [해당 사항 또는 "없음"]
  4. Evergreen Trigger: [해당 사항 또는 "없음"]
[⚠️ 경고 메시지 — Weak/None일 때만]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Step 3: 콘셉트 개발

제목과 썸네일을 **동시에 하나의 콘셉트 카드**로 설계. **총 10개** 생성.

콘셉트 배분은 **Step 0 판정 결과**에 따른다:

| 판정 | Drama | Strategy | Experimental |
|------|:-----:|:--------:|:------------:|
| Drama 우세 | 6 | 3 | 1 |
| Strategy 우세 | 3 | 6 | 1 |
| 하이브리드 | 4 | 4 | 2 |

### 썸네일 카피 11원칙 (v6 유지)

1. 제목과 **다른 레이어** (요약 금지)
2. **통념 뒤집기** or **인사이트 암시**
3. **시청자 참여** (질문, 도발, 자기 투영)
4. **구체성 필수** — 모호한 감정 태그 절대 금지
5. **영상 주제 키워드 포함**
6. **영상 전체 내용과 일치** — 30초 소재 확대 금지
7. **자기투영 가능해야 함** — "나한테 해당되나?"
8. **정보 밀도 높을수록 좋다** — 텍스트 + 그래프 + 태그가 서로 보강
9. **타겟 범위를 넓히는 단어 선택** — "Junior Dev" ✗ → "Software Engineer" ✓
10. **새로운 정체성/프레임워크 제안 > 실수 지적** — aspirational > guilt
11. **1초 모바일 가독성 테스트** — 모바일에서 엄지 스크롤 중 1초 안에 의미 전달

### Founder 추가 3원칙

12. **Before→After 숫자 대비** — 창업 전후의 극적 변화를 수치로 표현 ("$0 → $50M")
13. **인간적 포기/비용 암시** — 성공만 보여주지 않음. 희생의 무게를 느끼게
14. **시의성 키워드 반영** — Step 2.5에서 도출된 Timeliness 키워드 최소 2개 콘셉트에 반영

### 썸네일 비주얼 디렉션 (7종) — Founder 전용

| 비주얼 유형 | 강도 | 사용 조건 |
|------------|:---:|----------|
| **Before→After** | ★★★★★ | 극적 수치 변화가 있을 때 (가장 강력) |
| **Quote Extraction** | ★★★★★ | 게스트가 강렬한 한 문장을 말했을 때 |
| **Number Contrast** | ★★★★☆ | 두 숫자의 대비가 충격적일 때 ("$0 → $100M") |
| **Emotion Close-up** | ★★★★☆ | 드라마형 파운더, 감정 피크 순간이 있을 때 |
| **Timeline** | ★★★☆☆ | 시간 순서로 보여줘야 서사가 사는 경우 |
| **Decision Fork** | ★★★☆☆ | 갈림길 선택이 핵심 서사일 때 ("STAY or QUIT?") |
| **Method Card** | ★★★☆☆ | 전략형 파운더의 프레임워크가 핵심일 때 |

### 썸네일 카피 금지사항

- 모호한 감정 태그 ("YOUR JOB IS NEXT")
- 스테레오타입 비주얼 (개발도상국 = 빈곤)
- 맥락 없는 짧은 문구
- 타겟 한정 라벨 ("For First-Time Founders:")
- 업계 전문 용어 ("Series A Mechanics 101")
- 소모된 YouTube 공식 ("The Mistake X% Make")
- 실수 지적형 (Guilt framing)
- 철지난 트렌드
- 성공 포르노 ("This 25-Year-Old Makes $10M/Year" — 질투만 유발, 투영 없음)
- HR/업계 전문 톤 ("How to Find Product-Market Fit")

### Self-Projection 필수 태그

모든 콘셉트에 **시청자 내면 독백**을 작성해야 한다. "와 대단하다"로만 끝나면 **탈락**.

#### 5가지 투영 유형

| 투영 유형 | 설명 | 내면 독백 예시 |
|----------|------|---------------|
| **Decision Mirror** | "나도 비슷한 갈림길에 서 있다" | "나도 대기업 때려치고 창업할까 고민 중인데..." |
| **Skill Transfer** | "저 방법을 나도 쓸 수 있다" | "저 프레임워크로 우리 팀 의사결정 방식 바꿔봐야겠다" |
| **Age Comparison** | "저 나이에 저걸 했다고?" | "나는 30인데 아직 시작도 못했는데... 아직 늦지 않았나?" |
| **Fear Projection** | "나도 저렇게 될 수 있다/되면 어쩌지" | "나도 창업하면 집을 팔아야 하나..." |
| **Aspiration Mapping** | "나도 저런 사람이 되고 싶다" | "3년 안에 나도 저 정도 매출을 만들 수 있을까?" |

**검증 기준:**
- 각 콘셉트에 투영 유형 최소 1개 태그 + 내면 독백 1문장 필수
- "와 대단하다" / "대박이다" 류는 투영이 아니라 **감탄** → 탈락
- 투영은 반드시 **행동 가능성**(내가 뭘 할 수 있는가)이나 **비교 가능성**(나와 뭐가 다른가)을 포함해야 함

### 제목 작성 원칙

- 썸네일과 같은 영상처럼 읽혀야 함
- em dash(—) 사용 금지 → 콤마/마침표
- 약어 금지: "CMU" ✗ → "Carnegie Mellon" ✓
- 현재 직함만 사용
- 한 문장 원칙
- **100자 제한** (파이프+게스트명 포함, 80자 이내 이상적)
- **"From X to Y" 변환형 우선** (38% vs "How to" 29.9%)
- **직접 도발 > 중립 관찰** ("Aren't Ready" > "Do Differently")
- 파이프 뒤 기관명 — 썸네일에 있으면 제목에서 생략 가능

### 스테레오타입 회피 원칙

글로벌/개발도상국 앵글 시:
- **국가 이름 텍스트** — 글로브 위에 게스트가 직접 언급한 국가명만 배치
- 빈곤/먼지길 이미지 절대 금지 — EO의 aspirational 톤과 불일치
- 국가 > 도시 (게스트가 말한 단위 그대로)

## Step 3.5: 인트로 설계 + Three-Combo 정렬

### Three-Combo 원칙

```
썸네일 → "무엇에 관한 영상인가?" (1초 시각 약속)
    ↕ 같은 방향
제목   → "왜 이 영상을 봐야 하는가?" (구체적 설명/도발)
    ↕ 같은 방향
인트로 → "그 약속을 60초 안에 증명" (데이터/스토리로 증거)
```

### 인트로 설계 원칙

1. **60초 이내**
2. **두 축 구조: Fear Open → Aspiration Land**
3. **구체적 숫자 1개 이상**
4. **게스트의 직접 발언 활용** (파운더의 생생한 표현이 신뢰감)
5. **썸네일 약속을 15초 안에 터치**
6. **인트로 끝 = 영상의 핵심 프레임 선언**

### 인트로 구조 — Drama 변형

| 파트 | 시간 | 역할 |
|------|:---:|------|
| **HOOK** | 0-10초 | 가장 극적인 순간 (실패/위기/포기) 선 공개 |
| **EVIDENCE** | 10-35초 | 그 위기가 얼마나 심각했는지 숫자로 증명 |
| **TWIST** | 35-50초 | 반전: "그런데 지금은..." + 현재 성과 숫자 |
| **FRAME** | 50-60초 | 핵심 프레임 제시 ("이 사람이 다른 점은...") |

### 인트로 구조 — Strategy 변형

| 파트 | 시간 | 역할 |
|------|:---:|------|
| **HOOK** | 0-10초 | 반직관적 주장/통계 선 공개 |
| **EVIDENCE** | 10-35초 | 그 주장을 뒷받침하는 데이터/사례 |
| **TWIST** | 35-50초 | "대부분은 이렇게 하지만, 이 파운더는..." |
| **FRAME** | 50-60초 | 프레임워크/방법론 이름 제시 |

### 인트로 구조 — YC-Educational 변형 (Strategy 하위 옵션)

Strategy 콘셉트 중 **Founder Lesson Hook**을 사용할 때 적용. YC "Startup School" 스타일의 교육적 밀도 + EO 인터뷰 서사를 결합.

| 파트 | 시간 | 역할 |
|------|:---:|------|
| **HOOK** | 0-10초 | 파운더가 직접 배운 교훈을 숫자와 함께 선언 ("I burned $2M learning this one thing") |
| **EVIDENCE** | 10-30초 | 그 교훈이 나오기까지의 실패/시행착오 (서사 요소) |
| **FRAMEWORK** | 30-45초 | 교훈을 구조화한 프레임워크 제시 (YC 스타일: "3 rules", "the framework") |
| **FRAME** | 45-60초 | "이걸 알았으면 X를 피할 수 있었다" → 시청자 행동 유도 |

> **YC vs EO 차이 주의**: YC는 범용 조언("How to find PMF")이 가능하지만, EO는 반드시 **이 파운더의 구체적 경험**에서 추출해야 함. "He built a $100M company using 3 rules he learned from failing twice" ✓ / "3 Rules for Startups" ✗

### 1. 펀딩 금액 검증

| 검증 항목 | 규칙 |
|----------|------|

| 검증 항목 | 규칙 |

| 약어 | 전체 이름 표기 ("MIT" → "Massachusetts Institute of Technology" 또는 스크립트 원문 그대로) |

| 검증 항목 | 규칙 |
|----------|------|
